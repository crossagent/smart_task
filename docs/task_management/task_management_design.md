# 模块设计文档：`task_management` (项目与任务元数据管理)

## 1. 模块定义 (Module Definition)
`task_management` 是整个 Smart Task Hub 的“记忆中心”，负责维护所有实体（项目、活动、模块、任务及其依赖）的元数据。它的核心功能是将业务实体的静态定义转化为数据库中的持久化记录，并为外部观察者和执行者提供一致的查询视图。

## 2. 模块接口 (Module Interface)
- **输入 (Input)**：
    - 来自模型或人工录入的“实体增删改”请求（JSON 格式）。
    - 数据库存储的具体凭证（DB_HOST, DB_NAME, DB_USER 等环境变量）。
- **输出 (Output)**：
    - 完成数据库操作后的执行摘要字符串（例如：“Successfully processed task...”）。
    - 结构化的“任务上下文”（包含关联的项目名称、模块说明、迭代目标等全量元数据 JSON）。
    - 反馈到 MCP 的“错误消息”或“空查询响应”。

## 3. 模块流程 (Module Flow)
1. **连接建立**：利用 PostgreSQL 驱动库在收到请求时动态建立数据库会话。
2. **原子更新**：对于所有“写操作”（Upsert），利用数据库原生原子锁机制，在同一 ID 的记录存在时自动执行更新，不存在时执行插入。
3. **数据类型兼容**：在“读操作”返回数据之前，对复杂的 `datetime` 和 `Decimal` 类型进行截断与格式化，使其符合 JSON 传输标准。
4. **图结构存储**：不仅存储任务内容，同时负责物理存储任务间的 `depends_on`（依赖链条），为调度器提供推演依据。
5. **权限隔离**：预设了 SQL 读写黑白名单逻辑，防止未经授权的删除或结构变更。
